# **Infrastructure Security Watchdog**

## **Вступление** 
В данном личном проекте был использован за основу инструмент kottster.
Его основными возможностями являются:

- Полная интеграция с вашей базой данных
- Создание, удаление, перезапись ваших таблиц
- Сборка собственных дашбордов
- Создание кастомных таблиц с помощью React и JavaScript

## **Требования к использованию kottster**
- Ваша дб с проектами
- Подключение к kottster
- Деплой/Подключение к инструменту

## **Основная Цель работы**
Показать навыки и знания в развертывании и поддержании full-stack задач.

# **Основные инструменты**

### **Docker и Docker compose** 
Использовался для быстрой установки основных зависимостей,
которые требуется для дальнейшей загрузки kottster.

Создание постоянных volumes для сохранения своих работ при случайном сносе, сбоях, и
пересозданий контейнеров.

### **Ci/cd Github_actions** 
Был нужен для автоматической проверки кода на ошибки, а так же
для доставки кода в облако.

### **Terraform with digitalocean cloud** 
Является основой для нашего кода т.к позволяет автоматически
настроить подходящие сервера с усиленной защитой и основными dev и prod инструментами.
